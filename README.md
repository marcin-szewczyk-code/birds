# MyBirds

A simple web app for bird recognition (image + sound).

## Features

- bird list (grid / tiles)
- sound playback
- image preview

---

## Technology

The app works without a classical backend.

- HTML + CSS – structure and layout  
- JavaScript – application logic (rendering, interactions)  
- JSON – data source  
- PHP – thin layer for:
  - language switch (?lang=pl / ?lang=en)
  - basic HTML rendering (better SEO than pure SPA, Single Page Application)

---

## Project structure

- index.php – entry point (HTML generated by PHP)  
- style.css – layout and styles  
- app.js – application logic  
- data.json – bird data  

---

## How it works

1. index.php renders HTML and sets language (?lang=pl / ?lang=en)
2. Browser loads app.js
3. app.js loads data.json
4. app.js renders bird tiles
5. Click on a tile:
   - updates name
   - loads audio
   - loads image

All main logic runs in the browser.

## How to add a bird

Each object in data.json represents one bird:

{
  "id": "kos",
  "name": "Kos",
  "image_url": "...",
  "audio_url": "...",
  "image_page": "...",
  "audio_page": "...",
  "credit": "Wikimedia Commons"
}

---

## More stable variant

You can use local files instead of external URLs:

data/kos.jpg  
data/kos.ogg  

Then update paths in data.json.

You can also write a small script (Python or PHP) to:
- download files
- store them locally
- generate updated data-local.json

---

## Notes

- no application backend (no API, no database)
- no frameworks (plain JavaScript)
- static + client-side rendering
- easy to modify via data.json

---

## Why JavaScript (not PHP)

- the app runs in the browser
- no server-side processing is needed
- avoids hosting issues (cURL, SSL, API limits)
- easy to deploy as a static site

PHP is used only for language handling and SEO.
